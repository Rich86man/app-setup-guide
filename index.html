Your part: 3 steps

1

Make a free GitHub account

GitHub is the free service that stores the shared copy of your app's code. A free account is all you'll ever need.

  1. Go to github.com/signup
  2. Enter your email, then pick a password and a username (anything you like).
  3. When it asks about a plan, choose the Free plan.
  4. Check your email and click the link to confirm your account.

✎ Write it down

Save your GitHub username and password somewhere safe — you'll log in once more in Step 2, and that's about it.

2

Connect Base44 to GitHub

This is the magic step — it links your Base44 app to that shared copy so everything stays in sync.

  1. Open your app in Base44, like normal.
  2. Look for a GitHub option — it's usually under the app's settings or a “</> Code” / developer menu. (If you can't find it, screenshot the screen and send it to me.)
  3. Click Connect and sign in with the GitHub account you just made. Click Authorize when GitHub asks for permission.
  4. When it offers to create a repository, choose Private and let it create a new one. You can name it after your app.

🔒 Why “Private”?

A private repository means only you (and people you invite, like me) can see the code. The public can't. Choose Private and you're set.

3

Invite me, then tell me it's done

So I can help with the code, add me to your new repository:

  1. On GitHub, open your new repository.
  2. Click Settings → Collaborators → Add people.
  3. Add my GitHub username: Rich86man
  4. Text me that it's connected — and you're done! 🎉

✓ That's everything on your end

You never have to touch GitHub again unless you want to. Keep building in Base44 as usual.

How we'll make changes after that

You don't need to learn any of this — it's just so you know what's happening behind the scenes.

  1. I pull down the latest code from the shared GitHub copy onto my computer.
  2. I describe what we want in plain English to Claude Code — for example, “add a search box to the jobs page” — and it writes and edits the code for me.
  3. I send the changes back to GitHub.
  4. Base44 picks them up automatically and they show up in your live app — same as if they'd been made in Base44.

⚠ One small habit

To avoid us bumping into each other, let's not both edit at the same moment. If you're about to do a big editing session in Base44, give me a heads-up — and I'll do the same before I push changes. Day-to-day little tweaks are totally fine.

Common questions

Does this cost anything?

No. The GitHub account is free, and the connection to Base44 is part of your existing setup. Your Base44 plan stays whatever it is now.

Will my app or my Base44 editor change?

No. Your app runs and publishes exactly the same way. You keep using Base44 and its built-in AI whenever you want. We're only adding a second door into the same house.

What even is “GitHub” and a “repository”?

GitHub is a free website that safely stores copies of code. A “repository” (or “repo”) is just the folder that holds your app's code. Base44 keeps it perfectly in sync with your app for you.

Is my code safe and private?

Yes — that's why we choose a Private repository in Step 2. Only you and people you specifically invite can see it.

Do I have to learn to code or use “git”?

Not at all. Your only job is the 3 steps above. Everything technical happens on my side.

Can we undo this later?

Anytime. Disconnecting GitHub in Base44 leaves your app completely untouched.
