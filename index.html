Setup: 3 steps

1

Make a free GitHub account

GitHub is the free service that stores the shared copy of your app's code. A free account is all you'll ever need.

  1. Go to github.com/signup
  2. Enter your email, then pick a password and a username (anything you like).
  3. When it asks about a plan, choose the Free plan.
  4. Check your email and click the link to confirm your account.

✎ Write it down

Save your GitHub username and password somewhere safe — you'll use them again in the next two steps.

2

Connect Base44 to GitHub

This is the magic step — it links your Base44 app to that shared copy so everything stays in sync.

  1. Open your app in Base44, like normal.
  2. Look for a GitHub option — it's usually under the app's settings or a “</> Code” / developer menu.
  3. Click Connect and sign in with the GitHub account you just made. Click Authorize when GitHub asks for permission.
  4. When it offers to create a repository, choose Private and let it create a new one. You can name it after your app.

🔒 Why “Private”?

A private repository means only you (and anyone you choose to invite) can see the code. The public can't. Choose Private and you're set.

3

Set up Claude Code

Claude Code is the assistant you'll talk to in plain English to make changes. It runs on your own computer (Mac or Windows).

  1. Go to claude.com/product/claude-code and follow the install steps for your computer.
  2. Sign in with a Claude account (create one if you don't have it). Active use runs on a paid Claude plan.
  3. Open Claude Code and ask it, in plain English, to download your project — for example: “Clone my GitHub repository called [your-app-name] and open it.” It handles the rest.

✓ When in doubt, just ask it

Claude Code is built to be talked to like a person. If a step here is unclear, tell Claude Code what you're trying to do and it will guide you through it.

How you'll make changes

Once it's set up, improving your app looks like this — no code or technical steps required from you.

  1. Open your project in Claude Code.
  2. Describe what you want in plain English — for example, “add a search box to the jobs page” or “make the buttons green.” Claude Code writes and edits the code for you.
  3. Ask it to save and publish — say something like “save my changes and push them to GitHub.”
  4. Base44 picks them up automatically and they show up in your live app — exactly as if they'd been made in Base44.

⚠ One small habit

Don't edit the same thing in Base44 and Claude Code at the exact same moment — finish in one before switching to the other. Going back and forth between sessions is totally fine; doing both at once can cause a mix-up.

Common questions

Does this cost anything?

GitHub is free. Claude Code needs a Claude account — there's a way to try it, and regular use runs on a paid Claude plan. Your Base44 plan stays whatever it is now.

Will my app or my Base44 editor change?

No. Your app runs and publishes exactly the same way. You keep using Base44 and its built-in AI whenever you want. This just adds a second door into the same house.

What even is “GitHub” and a “repository”?

GitHub is a free website that safely stores copies of code. A “repository” (or “repo”) is just the folder that holds your app's code. Base44 keeps it perfectly in sync with your app for you.

Is my code safe and private?

Yes — that's why you choose a Private repository in Step 2. Only you and anyone you specifically invite can see it.

Do I need to learn to code or use “git”?

No. You describe what you want in plain English, and Claude Code does all the technical parts — including saving and publishing your changes.

Can I undo this later?

Anytime. Disconnecting GitHub in Base44 leaves your app completely untouched.
